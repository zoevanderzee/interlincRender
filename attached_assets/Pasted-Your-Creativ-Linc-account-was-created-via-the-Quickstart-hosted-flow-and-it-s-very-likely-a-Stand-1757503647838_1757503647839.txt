Your “Creativ Linc” account was created via the Quickstart/hosted flow, and it’s very likely a Standard (or otherwise not platform-controlled) Connect account. Embedded components only work for Express or Custom accounts you control.
If you pass a Standard acct_… into @stripe/connect-js, connect.create('account-onboarding') returns a non-mountable object → the exact “not mountable” error you’re seeing.
The dashboard screenshot (“Actions required”, “Payments paused”) doesn’t change that—Standard accounts can still have requirements, but you can’t embed the onboarding for them.
Fix (send this to your agent)
1) Guard the server: refuse Standard accounts and tell the client what to do
// POST /api/connect/create-account-session
app.post("/api/connect/create-account-session", async (req, res) => {
  try {
    const { accountId } = req.body || {};
    if (!accountId) return res.status(400).json({ error: "Missing accountId" });

    const acct = await stripe.accounts.retrieve(accountId);
    const type = acct.type; // "standard" | "express" | "custom"

    if (type === "standard") {
      // Embedded is not supported for Standard accounts
      return res.status(409).json({
        error: "embedded_not_supported_for_standard",
        message: "Embedded Connect components require an Express or Custom account.",
        accountType: type,
      });
    }

    // Decide if onboarding is still needed
    const r = acct.requirements || {};
    const needsOnboarding =
      !acct.details_submitted ||
      (Array.isArray(r.currently_due) && r.currently_due.length > 0) ||
      (Array.isArray(r.past_due) && r.past_due.length > 0);

    // Create an Account Session with both components enabled
    const session = await stripe.accountSessions.create({
      account: accountId,
      components: {
        account_onboarding: { enabled: true },
        account_management: { enabled: true },
      },
    });

    return res.json({
      accountId,
      accountType: type,
      needsOnboarding,
      client_secret: session.client_secret, // opaque string; do not pattern-check
    });
  } catch (e:any) {
    res.status(500).json({ error: e.message });
  }
});

Why it broke
You mixed an account created via Quickstart/hosted (probably Standard) with an embedded client. The embedded components correctly refused to mount because the account isn’t Express/Custom. Once you use an Express or Custom account ID, onboarding will mount and run inside your app.
