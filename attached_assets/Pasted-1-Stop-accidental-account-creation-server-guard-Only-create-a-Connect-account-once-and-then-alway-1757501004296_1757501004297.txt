1) Stop accidental account creation (server guard)
Only create a Connect account once and then always reuse it.
// server/connect-routes.js
app.post("/api/connect/create-account-session", async (req, res) => {
  try {
    const userId = req.user.id; // however you auth
    // 1) Look up existing mapping in your DB
    let accountId = await db.users.getConnectAccountId(userId);

    // 2) Create ONLY if explicitly allowed and missing
    const { allowCreate = false, country = "GB" } = req.body || {};
    if (!accountId) {
      if (!allowCreate) {
        return res.status(400).json({ error: "No Connect account for user; set allowCreate=true once to create." });
      }
      const acct = await stripe.accounts.create({
        type: "custom",
        country,
        capabilities: { card_payments: { requested: true }, transfers: { requested: true } },
      });
      accountId = acct.id;
      await db.users.setConnectAccountId(userId, accountId); // persist!
    }

    // 3) Always create an Account Session for the existing accountId
    const session = await stripe.accountSessions.create({
      account: accountId,
      components: { account_onboarding: { enabled: true } },
    });

    res.json({ accountId, client_secret: session.client_secret });
  } catch (e) {
    res.status(500).json({ error: e.message });
  }
});
Key idea: persist user.connectAccountId in your DB the first time. After that, never call stripe.accounts.create for that user again.
2) Client must pass/use the existing account
Fetch the user’s acct_… from your API/session and pass it—do not send null.
// client (pseudo)
const me = await fetch("/api/me").then(r=>r.json());        // { connectAccountId: "acct_..." }
const resp = await fetch("/api/connect/create-account-session", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    accountId: me.connectAccountId,   // ← reuse, don’t create new
    allowCreate: false,               // safety in prod; true only on the one-time setup path
  }),
});
Also keep your React effect guarded so it doesn’t run twice:
const started = useRef(false);
useEffect(() => { if (started.current) return; started.current = true; /* … */ }, []);
3) Don’t validate the secret by prefix
Delete any checks like startsWith("seti_"). Just ensure it’s a non-empty string and pass it to:
await loadConnectAndInitialize({ publishableKey: PK, fetchClientSecret: async () => client_secret });
4) Clean up the duplicates
Test mode: you can delete the extra accounts safely.
Live mode: if they have no activity, you can usually delete; otherwise “reject”/disconnect them. (Do this in Dashboard or via stripe.accounts.del(acctId) where allowed.)
Quick “no-more-fakes” acceptance checklist
 Your DB has a users.connectAccountId (string) column/field.
 First visit: server created one account, saved it, and returned it.
 Subsequent visits: server logs show no stripe.accounts.create calls for that user.
 /api/connect/create-account-session returns { accountId: "acct_…", client_secret: "<non-empty string>" }.
 The embedded onboarding renders inside your app (no redirect).
